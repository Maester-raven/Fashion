# 3.1.1 Model Provenance

This document records the origin and identity of the model asset used by Module 3.1.1 of the project **多模态驱动的电商服饰细粒度语义增强与智能解析系统**.

## 1. Base model source

- Base architecture: RTMDet-Ins-L
- Source family: OpenMMLab / MMDetection RTMDet series
- RTMDet paper release year: 2022
- Usage in this project: RTMDet-Ins-L is used as the main backbone/head family for fashion instance segmentation.

The checkpoint in this project is not a direct, unmodified RTMDet release checkpoint. It is a task-adapted model for e-commerce fashion parsing.

## 2. Data source

- Dataset family: Fashionpedia
- Fashionpedia paper release year: 2020
- Usage in this project: Fashionpedia categories are mapped into an 8-class e-commerce fashion taxonomy.
- Data handling policy: original data is not modified in place; the project generates derived manifests and intermediate files for training, validation, and deployment experiments.

## 3. Project-specific adaptation

The final 3.1.1 checkpoint is adapted through the project pipeline rather than being used as an original RTMDet model directly. The adaptation includes:

- 8-class fashion category remapping
- Fashionpedia data cleaning and COCO-format adaptation
- Class-specific threshold search
- Mask overlap conflict-resolution rules
- A-only TensorRT deployment path
- Clean-server deployment validation

## 4. Final 8 classes

1. top
2. pants
3. skirt
4. outerwear
5. dress
6. shoes
7. bag
8. accessory

## 6. Deployment asset policy

ONNX and TensorRT engine files are not uploaded as repository files. They should be regenerated locally from the checkpoint on the target machine because TensorRT engines are strongly coupled to GPU architecture, CUDA, TensorRT, and driver versions.
